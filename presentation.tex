% Page 2: Project Overview & Objectives
\begin{frame}{1. Project Overview \& Core Objectives}
    \textbf{The Scale \& The Challenge:} We engineered a high-performance sociodynamic pipeline to ingest and analyze \textbf{70GB of raw, scraped JSON payloads} spanning \textbf{83 distinct subreddits} and millions of unique user interactions. Reddit was modeled not as text, but as a massive directed multigraph where users are nodes and replies are weighted edges.
    
    \vspace{0.3cm}
    \textbf{Core Architectural Objectives:}
    \begin{itemize}
        \item \textbf{High-Throughput Sanitization:} Process 14,000,000+ comments iteratively via memory-efficient \texttt{ZStandard} streaming, overcoming the fatal Out-Of-Memory limits of traditional Pandas loading.
        \item \textbf{Topological Pruning:} Isolate the 25\% "conversational backbone" of communities by purging 75\% of noisy, fluff interactions via mathematical quantile thresholding.
        \item \textbf{Broker Identification:} Utilize Ronald Burt's Constraint Index to locate the $<0.1\%$ "Shadow Brokers"—users occupying structural holes bridging isolated echo chambers.
        \item \textbf{Epidemic Simulation:} Execute the Independent Cascade Model (ICM) to mathematically prove the accelerated viral propagation of rumors when seeded through Brokers.
    \end{itemize}
\end{frame}

% Page 3: Data Ingestion & Sanitization Engine
\begin{frame}{2. Data Mining, Ingestion \& Sanitization}
    \begin{columns}[T]
        \begin{column}{0.48\textwidth}
            \textbf{Data Mining via PRAW \& Zst Archiving} \\
            We mined Reddit using the Python Reddit API Wrapper (\textbf{PRAW}) to extract deeply nested conversational trees. To optimize storage, the massive payload was aggressively compressed and archived directly into \textbf{.zst} (ZStandard) NDJson dumps, resulting in a dense 70GB dataset.
            
            \vspace{0.2cm}
            \textbf{Memory-Efficient Streaming} \\
            Loading 70GB crashes conventional systems. We utilize an \texttt{io.TextIOWrapper} piped into a \texttt{zstd.stream\_reader} to process the archives line-by-line, executing $O(N)$ operations while consuming $<50\text{MB}$ of RAM.
        \end{column}
        \begin{column}{0.48\textwidth}
            \textbf{The "Anti-Fluff" Quantile Pruning} \\
            Over 80\% of social data is noise (bots, one-time replies). 
            \begin{itemize}
                \item We map edge weights by interaction frequency.
                \item We calculate the 75th percentile threshold.
                \item All edges below this limit are ruthlessly pruned.
            \end{itemize}
            \vspace{0.2cm}
            \textbf{Dynamic Entity Resolution:} Reddit anonymizes authors via IDs (\texttt{t1\_abc123}). We dynamically build $O(1)$ hash maps per chunk to resolve broken links into directed edges (\textit{User A $\rightarrow$ User B}).
        \end{column}
    \end{columns}
\end{frame}

% Page 4: Feature Normalization & Latent Modeling
\begin{frame}{3. Latent Modeling: Vectorizing Behavior (Technology Subreddit)}
    \begin{columns}[c]
        \begin{column}{0.55\textwidth}
            \textbf{Feature Scaling \& Normalization} \\
            Before modeling, topological metrics (Degree, Betweenness) must be scaled. We apply $Z$-score Normalization (\texttt{StandardScaler}) to enforce $\mu=0, \sigma=1$, ensuring "whales" with 1M+ Karma do not artificially warp the machine learning space.
            
            \vspace{0.3cm}
            \textbf{Node2Vec \& UMAP} \\
            How do we mathematically group cliques? 
            \begin{enumerate}
                \item \textbf{Node2Vec:} Executes 50 random walks per user, mapping their network position into a \textbf{64-dimensional vector}.
                \item \textbf{UMAP:} Human vision stops at 3D. UMAP flawlessly crushes these 64 dimensions into a 2D ($X, Y$) scatter plot, allowing us to visualize distinct, clustered echo chambers.
            \end{enumerate}
        \end{column}

        \begin{column}{0.55\textwidth}
            \textbf{The Scale-Free Reality} \\
            Our extracted graphs rigorously follow a \textbf{Power-Law Distribution} (left). 99\% of nodes have minimal connections, while a 1\% elite control thousands. This proves our parsed 70GB graph matches real-world human societal physics.
            
            \vspace{0.3cm}
            \textbf{Burt's Constraint Index} \\
            We locate users with mathematically minimal "Constraint" $C_i$:
            $$ C_i = \sum_j \left( p_{ij} + \sum_q p_{iq} p_{qj} \right)^2 $$
            Nodes scoring $<0.15$ are flagged as \textbf{Shadow Brokers}. Unlike standard popular nodes, Brokers sit directly inside the "structural holes" bridging isolated echo chambers, granting them ultimate information control.
        \end{column}
    \end{columns}
\end{frame}

% Page 6: Epidemic Simulation
\begin{frame}{5. Epidemic Simulation: The Viral Contagion}
    \begin{columns}[c]
        \begin{column}{0.55\textwidth}
            \textbf{The Independent Cascade Model (ICM)} \\
            To test the power of Shadow Brokers, we inject them with a simulated rumor and execute the ICM protocol.
            \begin{itemize}
                \item At $t=0$, we activate the top 5 Brokers.
                \item At each step, an infected node attempts to transmit to its neighbors with a $p=0.20$ probability.
            \end{itemize}
            
            \vspace{0.2cm}
            \textbf{The S-Curve Explosion} \\
            The resulting activation curve maps a classic epidemiological Contagion. It proves that strategically targeting Brokers shatters the network's normal flow, causing the information to go viral at a 300\% accelerated rate compared to random seeding.
        \end{column}
